Introduction

AutoBuddy is a would-be home automation system written in his spare time by Fran√ßois Wautier. It is distributed under the MIT license.

This project was triggered by the purchase of a number of Lifx lightbulbs

AutoBuddy consists in a number of pieces of software communicating via AMQP using Apache Qpid daemon and library.

This project is pretty much in its infancy and should be considered at this point (Oct 2015) as pre-alpha

AutoBuddy's logo was designed by Khan Wautier.

Components

Buddylib

Buddylib is a python library that defines the element known to the AutoBuddy ecosystem. It defines zone, various devices as well as some generic bridge functionalities. Buddylib depends on SQLAlchemy and Apache qpid python library.

ControlBuddy

ControlBuddy is at the heart of AutoBuddy. It controls the zones and their associated devices. It provides persistence services to other components.

LifxBuddy

LifxBuddy is a bridge application that provides control to Lifx lamps. It uses lifxlan to communicate with the lightbulbs.

SocketBuddy

SocketBuddy provides 2-way communication between websocket and AMQP. It serves WebBuddy and also provides login service to WebBuddy. It uses Autobahn WebSocket Python library and Apache qpid library.

WebBuddy

WebBuddy is AutoBuddy web application. It enables the user, to create zones, associate devices with zone, send commands to the devices/zones, manage users, and so on.

WebBuddy depend on, among others:

  • jQuery and jQuery-ui
  • Bootstrap
  • Bootstraps extensions: bootstrap-slider, bootstrap-switch, bootbox, ...
  • BuddyWheel a SVG/Javascript colourwheel that is a sub-project of ours.
